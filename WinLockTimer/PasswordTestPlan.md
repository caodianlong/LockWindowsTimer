# WinLockTimer 密码功能测试计划

## 测试目标
验证密码设置、保存、验证的完整流程，确保重启后密码验证正常工作。

## 测试场景

### 场景1：首次设置密码
1. 启动程序
2. 在密码文本框输入新密码（如：123456）
3. 点击"开始"按钮
4. 验证密码是否被正确保存
5. 重启程序
6. 验证密码文本框显示占位符"●●●●●●"
7. 尝试暂停/继续操作，验证密码验证是否正常工作

### 场景2：重启后密码验证
1. 在已有密码设置的情况下重启程序
2. 点击"暂停"按钮
3. 输入正确密码，验证是否通过
4. 输入错误密码，验证是否拒绝

### 场景3：修改密码
1. 点击"修改密码"按钮
2. 验证原密码（如果有）
3. 设置新密码
4. 验证新密码是否生效

### 场景4：清除密码
1. 点击"清除设置"按钮
2. 验证密码是否被正确清除
3. 重启程序验证密码状态

## 技术验证点

### 密码存储验证
- 设置文件应包含BCrypt哈希密码
- 密码不应以明文形式存储
- 重启后内存中的`parentPassword`应为BCrypt哈希

### 密码验证逻辑
- `VerifyPassword`方法应正确识别BCrypt哈希
- 使用`BCrypt.Net.BCrypt.Verify`进行密码验证
- 不应出现双重哈希问题

### 界面状态
- 已设置密码时显示占位符"●●●●●●"
- 未设置密码时显示空文本框
- 密码字符显示正确（● vs *）

## 预期结果
- 所有密码操作应正常工作
- 重启后密码验证应成功
- 无双重哈希问题
- 密码安全存储
